# Metrics Interpretation Guide

This document describes every field in the three metrics files generated per lesson.
Use it as a reference when building visualizations.

---

## File Map

```
data/processed/{student}/{lesson}/
  metrics/
    lesson_metrics.json    ← grammar + fluency, chunk-level + aggregate
    overall_metrics.json   ← combined score per chunk and lesson total
```

Generated by:
```bash
python src/extract-metrics/build_lesson_metrics.py  [student lesson]
python src/extract-metrics/build_overall_metrics.py [student lesson]
```

---

## Score Labels & Colors

All three files share the same label/color table:

| score range | label    | color     | meaning                                 |
|-------------|----------|-----------|-----------------------------------------|
| ≥ 70        | Rich     | `#15803d` | Strong performance                      |
| 50 – 69     | Moderate | `#d97706` | Solid but room to grow                  |
| 30 – 49     | Basic    | `#ea580c` | Clear gaps, foundational level          |
| < 30        | Sparse   | `#dc2626` | Very limited                            |

Fluency sentences use a slightly different table:

| score range | label     | color     |
|-------------|-----------|-----------|
| ≥ 80        | Fluent    | `#15803d` |
| 60 – 79     | Moderate  | `#d97706` |
| 40 – 59     | Struggling| `#ea580c` |
| < 40        | Weak      | `#dc2626` |

Every scored object carries three fields: `score` (float 0–100), `score_label` (string), `color` (hex).

## 1 · lesson_metrics.json

Top-level keys: `student`, `lesson`, `generated_at`, `grammar`, `fluency`.

---

### 1.1 Grammar section (`lesson_metrics.grammar`)

#### Formula

```
level_score   = richness.level   × 40        → 0 – 40
variety_score = richness.variety × 20        → 0 – 20
error_score   = (error_quality / 100) × 40  → 0 – 40
chunk_score   = level_score + variety_score + error_score  → 0 – 100
```

**Partial score**: if either grammar richness or errors data is missing, the available
weights are re-scaled so the total still reaches 100. The chunk and aggregate are
both marked `partial: true`.

| field              | type          | description                                                    |
|--------------------|---------------|----------------------------------------------------------------|
| `has_grammar`      | bool          | grammar_richness.json was found for this lesson                |
| `has_errors`       | bool          | errors.json was found for this lesson                          |
| `partial`          | bool          | at least one source file is missing                            |
| `formula`          | object        | static weights used (level_weight=40, variety_weight=20, error_weight=40, min_chunk_words=10) |

#### grammar.aggregate

| field              | type          | description                                                     |
|--------------------|---------------|-----------------------------------------------------------------|
| `score`            | float 0–100   | mean grammar score across included chunks                       |
| `score_label`      | string        | Rich / Moderate / Basic / Sparse                                |
| `color`            | hex string    | display color for the label                                     |
| `level_score`      | float 0–40    | mean level component across included chunks                     |
| `variety_score`    | float 0–20    | mean variety component across included chunks                   |
| `error_score`      | float 0–40 \| null | mean error component (null if errors.json missing)         |
| `chunks_total`     | int           | total paragraphs in the lesson                                  |
| `chunks_included`  | int           | paragraphs used for scoring (≥ 10 words)                        |
| `chunks_skipped`   | int           | paragraphs too short to score                                   |
| `partial`          | bool          | true if any source is missing                                   |

#### grammar.chunks[] — one entry per paragraph

| field              | type          | description                                                     |
|--------------------|---------------|-----------------------------------------------------------------|
| `chunk_id`         | int           | paragraph_id from sentences.json (not always contiguous)        |
| `label`            | string        | auto-generated topic label for this paragraph                   |
| `word_count`       | int           | approximate word count (summed from sentence texts)             |
| `sentence_count`   | int           | number of sentences in the paragraph                            |
| `skipped`          | bool          | true if word_count < 10                                         |
| `skip_reason`      | string \| null | reason for skipping, e.g. "too short (7w < 10)"               |
| `score`            | float \| null | grammar score 0–100 for this chunk                              |
| `score_label`      | string \| null | label for the score                                            |
| `color`            | hex \| null   | color for the score                                             |
| `level_score`      | float 0–40    | CEFR level component for this chunk                             |
| `variety_score`    | float 0–20    | category variety component for this chunk                       |
| `error_score`      | float 0–40 \| null | error quality component (null if errors.json missing)      |
| `has_grammar`      | bool          | richness data exists for this chunk                             |
| `has_errors`       | bool          | error data exists for this chunk                                |
| `partial`          | bool          | true if any source is missing for this chunk                    |
| `grammar`          | object \| null | grammar richness detail (see below)                            |
| `errors`           | object \| null | error detail (see below)                                       |

#### grammar.chunks[].grammar — richness detail

| field                | type     | description                                                    |
|----------------------|----------|----------------------------------------------------------------|
| `richness_score`     | int 0–100 | original richness score (old formula: level×60 + variety×40) |
| `level`              | float 0–1 | normalized average CEFR level: `(avg_level_numeric − 2) / 4` |
| `level_score`        | float 0–40 | `level × 40`                                                 |
| `variety`            | float 0–1 | `distinct_categories / 15` (capped at 1)                     |
| `variety_score`      | float 0–20 | `variety × 20`                                               |
| `avg_cefr`           | string   | average CEFR level as label: A1 / A2 / B1 / B2 / C1 / C2    |
| `n_assigned`         | int      | number of grammar structures matched (after de-overlap)       |
| `density`            | float    | `n_assigned / sentence_count` — matches per sentence          |
| `distinct_categories`| string[] | grammar categories present (e.g. VERBS, CLAUSES, PRONOUNS…)  |
| `dims_present`       | string[] | Proficiency Dimensions present: A / B / C / D                 |
| `level_distribution` | object   | `{ "A2": 29, "B1": 24, "B2": 2 }` — count per CEFR level    |
| `top_match`          | object   | highest-level structure matched in this chunk (see below)     |

**top_match fields**: `structure_id`, `category`, `guideword`, `lowest_level`,
`lowest_level_numeric`, `explanation`, `span_text`, `sentence_text`,
`start_char`, `end_char`, `context_start_char`, `context_end_char`.

#### grammar.chunks[].errors — error detail

| field                | type     | description                                                    |
|----------------------|----------|----------------------------------------------------------------|
| `error_count`        | int      | number of LanguageTool errors detected in this chunk           |
| `weighted_error_sum` | int      | sum of error weights (weight 1–3 per error)                    |
| `quality_score`      | int 0–100 | `max(0, 100 − 5 × weighted_sum / word_count) × 100`          |
| `quality_level`      | string   | Good / Moderate / Weak / Poor                                  |
| `error_score`        | float 0–40 | `(quality_score / 100) × 40`                                 |
| `dimension_counts`   | object   | `{ "A": 0, "B": 2, "C": 0, "D": 10 }` — errors by dimension  |

**Error weight scale**: 1 = beginner error (green), 2 = medium (amber), 3 = high (red).
**Proficiency Dimensions**: A = Sentence Architecture, B = Tense & Aspect, C = Nominal Precision, D = Modal & Functional Range.

### 1.2 Fluency section (`lesson_metrics.fluency`)

Each sentence score is computed from four components:

```
speed_component   = how fast the learner articulates words (sec/letter, lower = faster)
gaps_component    = ratio of pause time to total speaking time (lower pauses = better)
fillers_component = proportion of filler / hesitation words (lower = better)
dups_component    = proportion of repeated words/phrases (lower = better)

sentence_score = 0.35·speed + 0.35·gaps + 0.15·fillers + 0.15·dups  →  0–100
```

Sentences with fewer than 5 words are excluded from scoring.

#### fluency.aggregate

| field                  | type        | description                                                  |
|------------------------|-------------|--------------------------------------------------------------|
| `score`                | float 0–100 | mean fluency score across included sentences                 |
| `score_label`          | string      | Fluent / Moderate / Struggling / Weak                        |
| `color`                | hex string  | display color                                                |
| `components.speed`     | float 0–100 | mean speed component (100 = very fast articulation)          |
| `components.gaps`      | float 0–100 | mean gaps component (100 = no thinking pauses)               |
| `components.fillers`   | float 0–100 | mean fillers component (100 = no fillers detected)           |
| `components.dups`      | float 0–100 | mean duplicates component (100 = no repetitions)             |
| `distribution.fluent`  | int         | sentences with score ≥ 80                                    |
| `distribution.moderate`| int         | sentences with score 60–79                                   |
| `distribution.struggling`| int       | sentences with score 40–59                                   |
| `distribution.weak`    | int         | sentences with score < 40                                    |
| `sentences_included`   | int         | sentences used (word_count ≥ 5, score not null)              |
| `sentences_skipped`    | int         | sentences excluded (too short or no score)                   |
| `sentences_total`      | int         | all sentences in the lesson                                  |
| `stats.total_words`    | int         | total word count in the lesson (all sentences)               |
| `stats.filler_count`   | int         | total filler words across the lesson                         |
| `stats.filler_rate`    | float 0–1   | `filler_count / total_words`                                 |
| `stats.filler_types`   | object      | counts by filler type: backchannel / hesitation / discourse_marker / placeholder |
| `stats.avg_speed_ms`   | int         | mean articulation speed in ms/letter (content words only)    |
| `stats.avg_gap_ms`     | int         | mean inter-word gap in ms across all sentences               |

#### fluency.sentences[] — one entry per included sentence (word_count ≥ 5)

| field               | type        | description                                                   |
|---------------------|-------------|---------------------------------------------------------------|
| `sentence_id`       | int         | 1-based sentence index from spaCy segmentation               |
| `text`              | string      | full sentence text                                            |
| `word_count`        | int         | number of words in the sentence                               |
| `score`             | float 0–100 | fluency score for this sentence                               |
| `score_label`       | string      | Fluent / Moderate / Struggling / Weak                         |
| `color`             | hex string  | display color                                                 |
| `components.speed`  | float 0–100 | articulation speed component                                  |
| `components.gaps`   | float 0–100 | inter-word gaps component                                     |
| `components.fillers`| float 0–100 | filler rate component                                         |
| `components.dups`   | float 0–100 | repetition component                                          |
| `filler_count`      | int         | number of filler words in this sentence                       |
| `filler_rate`       | float 0–1   | `filler_count / word_count`                                   |
| `gap_mean_ms`       | float \| null | mean inter-word gap in ms (null if only 1 word)             |
| `duplicate_count`   | int         | extra word occurrences beyond first (e.g. "I was I was" → 2) |
| `accuracy_mean`     | float \| null | mean ASR confidence score 0–1 for words in this sentence   |

**Note on sentence alignment**: fluency sentences come from spaCy segmentation of the
full lesson ASR transcript, which may differ from the paragraph sentence count in
`grammar_richness.json`. They are aligned to grammar chunks in `overall_metrics.json`
using cumulative word-count position.

## 2 · overall_metrics.json

Combines grammar and fluency into a single score per chunk and lesson.

### Formula

```
overall_score  = grammar_score × (fluency_score / 100)   →  0 – 100
lesson_overall = mean(chunk_overall_scores)
```

**Fluency score per chunk**: mean of sentence fluency scores (≥5 words) whose
cumulative word position falls within the chunk's `word_start_id`–`word_end_id` range.
If fewer than 3 valid sentences are mapped to a chunk, the lesson-level fluency mean
is used as fallback (`fluency_fallback: true`).

**Future extension**: when vocabulary metrics are available, `grammar_score` will become
`mean(grammar_score, vocab_score)`. The `overall_score` formula stays identical.

### overall_metrics.aggregate

| field               | type          | description                                                    |
|---------------------|---------------|----------------------------------------------------------------|
| `score`             | float \| null | mean of chunk overall_scores (null if fluency missing)         |
| `score_label`       | string        | Rich / Moderate / Basic / Sparse (or N/A)                      |
| `color`             | hex string    | display color                                                  |
| `grammar_score`     | float         | mean grammar_score across included chunks                      |
| `fluency_score`     | float \| null | mean per-chunk fluency score across included chunks            |
| `chunks_included`   | int           | chunks used in aggregation                                     |
| `chunks_skipped`    | int           | chunks excluded (word_count < 10)                              |
| `chunks_total`      | int           | total paragraph count                                          |
| `grammar_partial`   | bool          | errors.json was missing (grammar re-scaled to 60 max)          |
| `fluency_partial`   | bool          | fluency.json was missing (overall_score = null)                |
| `partial`           | bool          | either grammar_partial or fluency_partial is true              |

### overall_metrics.chunks[] — one entry per paragraph

| field                        | type          | description                                                |
|------------------------------|---------------|------------------------------------------------------------|
| `chunk_id`                   | int           | paragraph_id                                               |
| `label`                      | string        | topic label for this paragraph                             |
| `word_count`                 | int           | approximate word count                                     |
| `sentence_count`             | int           | number of sentences                                        |
| `skipped`                    | bool          | true if word_count < 10                                    |
| `skip_reason`                | string \| null | reason if skipped                                         |
| `grammar_score`              | float \| null | grammar score 0–100 for this chunk                         |
| `grammar_partial`            | bool          | grammar score was re-scaled (errors.json absent)           |
| `fluency_score`              | float \| null | mean fluency score of sentences in this chunk              |
| `fluency_sentences_included` | int           | sentences (≥5 words) mapped to this chunk                  |
| `fluency_fallback`           | bool          | lesson-level fluency score was used (< 3 sentences in chunk)|
| `overall_score`              | float \| null | `grammar_score × (fluency_score / 100)`                    |
| `score_label`                | string \| null | label for the overall score                               |
| `color`                      | hex \| null   | display color                                              |
| `partial`                    | bool          | grammar_partial or fluency missing                         |

## 3 · Worked Example

**Student-1 / lesson-1 / chunk 1**

```
Grammar (partial — no errors.json):
  level   = 0.127  → level_score   = 0.127 × 40 = 5.1
  variety = 0.800  → variety_score = 0.800 × 20 = 16.0
  Re-scaled to 100: score = (5.1 + 16.0) × (100/60) = 35.2

Fluency (14 sentences ≥5w in this chunk):
  mean sentence score = 64.4

Overall:
  35.2 × (64.4 / 100) = 22.7
```

**Lesson aggregate (9 chunks):**

```
grammar aggregate  = mean([35.2, 11.8, 20.2, 9.0, 23.9, 17.1, 25.1, 27.3, 34.0]) = 22.6
fluency aggregate  = mean([64.4, 53.0, 55.8, 63.0, 63.5, 69.7, 67.5, 67.7, 58.6]) = 62.6
overall aggregate  = mean([22.7,  6.3, 11.3,  5.7, 15.2, 11.9, 16.9, 18.5, 19.9]) = 14.3
```

## 4 · Partial Score Handling

A score is **partial** when a source file is missing:

| situation                           | grammar_score behavior             | overall_score behavior        |
|-------------------------------------|------------------------------------|-------------------------------|
| grammar + errors both present       | full score out of 100 (40+20+40)   | grammar × fluency / 100       |
| grammar only (no errors.json)       | re-scaled: (level+variety)×100/60  | same formula, but partial=true |
| errors only (no grammar_richness)   | re-scaled: error×100/40            | same formula, but partial=true |
| fluency.json missing                | grammar computed normally          | overall_score = null           |

When `partial: true`, scores are still comparable within the same lesson across chunks,
but **cross-lesson comparisons should only use lessons where `partial` is false for both**.

## 5 · Visualization Recipes

### Bar chart — grammar breakdown per chunk
Fields: `grammar.chunks[].chunk_id`, `.level_score`, `.variety_score`, `.error_score`
Stacked bar: level (blue, max 40) + variety (green, max 20) + errors (red, max 40).
Use `.label` as axis label. Grey out skipped chunks.

### Scatter plot — grammar vs fluency per chunk
X axis: `overall_metrics.chunks[].grammar_score` (0–100)
Y axis: `overall_metrics.chunks[].fluency_score` (0–100)
Point size: `word_count`. Color: `color` field.
Shows whether grammar richness and fluency co-vary or diverge.

### Lesson-level summary card
`overall_metrics.aggregate.score` — headline number.
`overall_metrics.aggregate.grammar_score` — sub-number.
`overall_metrics.aggregate.fluency_score` — sub-number.
Use `color` for the badge background.

### Fluency sentence timeline
X axis: `lesson_metrics.fluency.sentences[].sentence_id`
Y axis: `lesson_metrics.fluency.sentences[].score`
Color per point: `color` field.
Overlay `filler_count > 0` as orange dots, `duplicate_count > 0` as red.

### Filler breakdown donut
`lesson_metrics.fluency.aggregate.stats.filler_types` — keys as slices.
Colors: backchannel `#3B82F6`, hesitation `#8B5CF6`, discourse_marker `#F97316`, placeholder `#EF4444`.

### CEFR level distribution bar (per chunk)
`grammar.chunks[].grammar.level_distribution` — e.g. `{ "A2": 29, "B1": 24, "B2": 2 }`.
Horizontal stacked bar. Colors: A1 `#9CA3AF`, A2 `#60A5FA`, B1 `#34D399`, B2 `#FBBF24`, C1 `#F97316`, C2 `#EF4444`.

### Grammar category heatmap
Rows: chunks. Columns: distinct grammar categories (VERBS, CLAUSES, PRONOUNS…).
Cell value: whether the category appears in `grammar.chunks[].grammar.distinct_categories`.
Useful for spotting which categories a student consistently avoids.

### Cross-lesson progression (when multiple lessons available)
One data point per lesson from `overall_metrics.aggregate`:
  `score`, `grammar_score`, `fluency_score`
Overlay all three as separate lines on a time axis (lesson-1, lesson-2, lesson-3…).
Mark `partial: true` lessons with a dashed line or warning icon.
